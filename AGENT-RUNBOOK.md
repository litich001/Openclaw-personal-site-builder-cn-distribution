# AGENT RUNBOOK

这份文件是写给自动化代理的执行说明。

目标不是解释“为什么”，而是尽量把执行路径固定下来，让代理在面对不同用户资料时也能稳定推进。

## 默认目录约定

推荐把用户项目整理成下面的结构：

```text
project-root/
├─ materials/
│  ├─ resume.pdf
│  ├─ intro.txt
│  ├─ links.txt
│  ├─ photos/
│  ├─ posters/
│  └─ certs/
├─ output-site/
│  ├─ site/
│  ├─ deploy/
│  └─ archives/
└─ notes/
```

建议含义如下：

- `materials/`
  放用户原始资料。
- `output-site/site/`
  放最终静态网站文件。
- `output-site/deploy/`
  放部署说明、平台信息、线上地址记录。
- `output-site/archives/`
  放可分享压缩包。
- `notes/`
  放问答补齐记录、文案决策、风格说明等。

如果用户已有自己的目录结构，可以沿用，但代理应先识别清楚资料位置再执行。

## 执行顺序

1. 扫描资料目录。
2. 判断资料是否完整。
3. 如果资料不完整，先进入问答补齐。
4. 把内容整理成网站结构。
5. 生成首版站点。
6. 检查桌面端、移动端、SEO、资源引用。
7. 输出静态目录。
8. 若具备权限，执行部署。
9. 记录真实线上地址并回填 SEO 字段。
10. 输出最终交付说明。

## 资料完整性判断

至少检查这些内容是否存在：

- 姓名
- 目标岗位
- 核心介绍
- 职业经历
- 项目经历
- 联系方式

如果缺少其中任意关键项，不要直接硬写完整网站，而是优先问答补齐。

## 问答补齐最小清单

如果资料不完整，至少补齐：

1. 想应聘什么岗位
2. 有多少年经验
3. 最想突出哪 3 个优势
4. 最重要的 2 到 4 段经历
5. 是否愿意公开电话、邮箱、微信
6. 更偏什么风格：专业稳重、科技感、创意型

## 输出要求

代理默认应输出：

- 首页
- 联系页或联系区
- 资料展示区
- 网页版简历页（推荐）
- 基础 SEO 文件

默认静态站结构至少包括：

```text
site/
├─ index.html
├─ resume.html
├─ styles.css
├─ script.js
├─ robots.txt
├─ sitemap.xml
├─ site.webmanifest
└─ assets/
```

## 设计收口要求

每轮调整后都要检查这些问题：

- 是否出现大片无意义空白
- 是否出现标题断词
- 是否出现桌面端左右失衡
- 是否出现移动端按钮挤压
- 是否出现过大的二维码卡片
- 是否出现相互冲突的重复媒体查询

如果发现样式开始失控，应先收口现有规则，再继续迭代。

## 部署默认策略

默认优先：

1. 输出可直接上传的静态目录
2. 优先部署到 Cloudflare Pages 或等价静态托管

如果代理具备上传能力，则继续执行部署。
如果不具备上传能力，则输出清晰的人工上传步骤。

## 部署回填规则

部署成功后，必须检查并同步：

- `canonical`
- `og:url`
- `sitemap.xml`
- 任何写死的预设线上地址

如果部署地址与预设地址不一致，必须回填真实线上地址。

## 失败时的回退路径

如果遇到问题，按下面顺序回退：

1. 无法读取资料
  回退到问答补齐模式
2. 无法自动部署
  输出静态目录和人工上传步骤
3. 无法访问部署平台
  记录失败原因，保留打包产物
4. 用户资料过少
  先生成骨架版网站，等待补充

## 最终交付清单

完成后至少交付：

- 静态网站目录
- 可分享压缩包
- 部署说明
- 真实线上地址
- 已回填后的 SEO 地址说明
